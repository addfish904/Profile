為了提升自己的問題解決能力，我養成了記錄錯誤與尋找解決方法的習慣，針對學習過程中遇到的困難，我會利用網路查詢資料或參考技術論壇的內容，必要時也會嘗試閱讀相關的官方文件，這些努力不僅幫助我加深對 JavaScript 的理解，也讓我遇到困難時不再手足無措，而是能夠冷靜地找到解決方法。

雖然目前的學習進度還在起步階段，但我對自己的學習能力充滿信心，我相信只要保持專注與持續努力，技術能力一定會逐漸提升，如果有機會參與本次計畫，我預計離開現有工作，投入全心全意學習，並以高度的自律與積極的態度迎接挑戰，讓自己成為一名具備實力與潛力的軟體工程師。

Time

如果參與這個訓練，會怎麼安排學習時間？

如果有幸入選這個訓練計畫，我會全心全意投入學習，為了確保自己能夠專注於課程內容，我會選擇離職，專注於課程要求與學習目標的完成，我深知這次機會對我未來職業發展的重要性，因此會以高度的自律與責任心規劃每一天的學習，為了提高學習效率，我會將一天以時間區分為三個階段，不同階段有不同的主要目標，讓學習更具系統性與效率。

早上是吸收新知識的關鍵時間段，我會在每週一的早上參加線上會議，了解當週的學習目標與任務，隨後開始一週的學習。每天早上，我會專注在學習核心內容，例如觀看教學影片、閱讀相關資料或參考文件，並記錄過程中遇到的重點和不理解的問題。在這段時間，我會保持專注，避免分心，確保自己能有效吸收新知識，我也會運用筆記工具整理內容，如Notion，幫助加深記憶並為之後的複習做好準備。

下午則是補充與實踐的階段。這段時間，我會深入研究早上記錄的問題，透過搜尋相關資料、參考技術論壇或文件進一步補足自己的理解，此外，我會試著將所學的內容運用到實際練習中，例如完成當週任務、練習程式碼或模擬實際應用情況，在實踐的過程中，如果遇到困難，我會先嘗試靠自己找出解決方案，讓自己培養獨立解決問題的能力，同時，我會記錄無法解決的難題，準備詢問助教，或在 Discord 發問和同學進行討論，確保問題能徹底解決並理解。

晚上則是回顧與整理的時間。在這個階段，我會回顧當天以及前一天的學習內容，確認自己的理解程度，並針對未解的問題再次進行思考。如果下午的實作過程中有不足的地方，我也會利用這段時間進一步優化改善。此外，我會整理當天的筆記，將零散的重點進行統整，以便複習和查閱。最後，我會撰寫當天的學習日誌，記錄當天完成的進度、學習的內容、遇到的困難，以及需要改進的方向，這不僅能幫助我檢視自己的學習狀態，還能作為持續調整學習計劃的依據。

總體來說，我希望通過這樣的學習安排，最大化每一天的價值，無論是吸收新知識，還是解決實際問題，我相信，持續而系統的學習方法，能讓我在訓練中充分掌握每一項技能，為轉職的路上打下堅實的基礎。

  • Q.請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

    前段時間，我嘗試將開源生成式 AI 模型 Stable Diffusion 結合繪圖軟體，實現 AI 即時繪圖功能，這個功能不僅可以讓AI協助你畫圖，還可以實現快速合成及3D繪畫，可以幫助我提高工作上的效率。然而安裝環境的過程中需要用到許多我不熟悉的技術，如 Git 和 Python，因此在過程中頻繁遇到各種錯誤，每次解決一個問題，又會冒出更多新的錯誤。不僅僅是技術層面的問題，還有 Stable Diffusion 本身較高的入門門檻，需要深入理解節點的概念、生成模型的工作原理以及各參數的配置方式，這些技術要求對於當時的我來說相當陌生，讓整個過程顯得格外艱難，種種挑戰累積下來，使我一度感到非常挫折，有很多人會在此階段放棄，轉而使用更方便，但操作靈活性也較低的 AI 工具，例如 Midjourney。
    雖然一度感到沮喪，但我決定不被負面情緒主導，給自己更多時間逐步克服挑戰。為了調整心態，我選擇暫時放下問題，離開電腦，透過運動或散步讓自己從壓力中抽離，幫助大腦放鬆，以更好的身心狀態回來處理問題。重新整理思緒後，我先嘗試理解問題的來源，並逐步拆解，我開始學習 Git 指令，熟悉版本控制的基本操作，並通過網路和技術社群尋找解決方法，我查閱了 Stable Diffusion 的官方文件和許多國外用戶的文章，逐步理解它的運作原理以及如何完成即時繪圖的設置。在處理錯誤時，我採取了更有條理的方式：完成一個環節後再解決下一個問題，避免一次性投入過多精力而導致挫敗感加重。最後我成功解決所有錯誤，並完成了環境設置！這次經歷讓我更深刻體會到，面對複雜問題時，最重要的是保持冷靜，靈活運用資源，並以正向的心態去應對挑戰。

  • Q.關於這份申請網頁，分享一個開發時的技術心得。

    在過去的切版經驗中，我大部分都是負責靜態網頁的切版，複雜的動態和功能則交由前端工程師處理，而在這次的申請網頁中，我嘗試加入一些利用 JavaScript 實現的動態效果，例如堆疊卡片、滑動浮現等，來提升網頁的互動性與視覺吸引力，這也是我在未來想達成的目標之一。過程中，我花了很多時間研究 JavaScript 的語法與應用，雖然網路上有許多豐富的資源可以參考，但實際將效果套用到網頁上時，仍需要根據情況進行多次調整和測試，例如在製作堆疊卡片的功能時，我遇到了卡片滾動卡住的問題， 當時我首先確認了 JavaScript 的執行邏輯，並使用 console.log 將關鍵變數的值打印出來，逐步檢查每一個函式的執行情況。一開始，我懷疑問題可能出在 addEventListener 的設定上，但排查後發現事件確實被正確觸發，因此轉而檢查與動畫效果相關的 CSS。 我使用瀏覽器的開發者工具，仔細觀察卡片的樣式變化，並發現卡片的 transform 屬性值未能如預期更新，接著我回到 JavaScript 中，追蹤函式中與樣式更新相關的部分，發現一個關鍵的 CSS 變數未被正確賦值，導致了動態的效果失效，進一步檢查後，我發現這是因為變數的名稱拼寫錯誤，導致瀏覽器無法解析。 修正了 CSS 變數的名稱後，我重新測試了功能，卡片滾動的問題就解決了！這個過程雖然耗時，但透過 console.log 和開發者工具逐步排查錯誤來源，讓我更加理解程式碼的運作邏輯，也學到了在面對複雜問題時如何有效拆解並解決。
    這次的經歷讓我深刻體會到，雖然學習技術的過程中可能會遇到很多困難，但只要以耐心與積極的態度面對，並學會運用系統化的解決方式，逐步克服每一個困難，最終伴隨的也是極大的成就感，我享受這種過程，也明白未來會遇到更多更巨大的挑戰，但我會以這次的經驗為基礎，秉持相同的態度，逐步分析問題並找到解決方法，解決每一個難關！
